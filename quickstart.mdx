## Get started with Smart Feedback API

This guide will walk you through installing, running, and making your first request to the Smart Feedback API.

<Steps>
  <Step title="Set up a virtual environment">
    Create and activate a Python virtual environment to isolate your dependencies:

    <CodeGroup>
    ```bash Linux/Mac
    python -m venv venv
    source venv/bin/activate
    ```

    ```bash Windows
    python -m venv venv
    venv\Scripts\activate
    ```
    </CodeGroup>

    <Note>
    Using a virtual environment is recommended to avoid conflicts with other Python projects.
    </Note>
  </Step>

  <Step title="Install dependencies">
    Install all required packages from the requirements file:

    ```bash
    pip install -r requirements.txt
    ```

    This installs the core dependencies:
    - FastAPI (0.128.5)
    - Uvicorn (0.40.0)
    - scikit-learn (1.8.0)
    - pandas (3.0.0)
  </Step>

  <Step title="Start the server">
    Launch the API server using Uvicorn:

    ```bash
    uvicorn app.main:app --reload
    ```

    The `--reload` flag enables auto-reloading during development. You should see output like:

    ```
    INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
    INFO:     Started reloader process
    INFO:     Started server process
    INFO:     Waiting for application startup.
    INFO:     Application startup complete.
    ```

    <Warning>
    The server needs access to `data/reviews.csv` to train the model on startup. Make sure this file exists in your project directory.
    </Warning>
  </Step>

  <Step title="Make your first request">
    Send a POST request to the `/analyze` endpoint:

    ```bash
    curl -X POST "http://127.0.0.1:8000/analyze" \
      -H "Content-Type: application/json" \
      -d '{"text": "¡Hola equipo! Quería compartir mi experiencia con el proceso de onboarding. La verdad es que fue mucho más fácil de lo que me imaginaba. Desde el primer día, me sentí muy bienvenido y las herramientas que proporcionaron fueron super útiles. Las sesiones de capacitación estaban bien estructuradas y los tutores eran muy pacientes; respondieron a todas mis dudas sin dudarlo. Además, la comunicación fue clara y siempre estuve al tanto de lo que seguía. Siento que tengo una buena base para empezar en este nuevo rol. Creo que hicieron un gran trabajo al diseñar todo, ¡mil gracias!"}'
    ```

    You'll receive a response with the sentiment analysis:

    ```json
    {
      "sentiment": "positivo",
      "score": 0.9498749406711353
    }
    ```

    The response includes:
    - `sentiment` - Detected sentiment (positivo, neutral, or negativo)
    - `score` - Confidence level from 0.0 to 1.0 (higher means more confident)
  </Step>

  <Step title="Explore the interactive docs">
    Open your browser and navigate to the auto-generated API documentation:

    ```
    http://127.0.0.1:8000/docs
    ```

    FastAPI automatically generates interactive documentation where you can:
    - View all available endpoints
    - See request/response schemas
    - Test API calls directly in your browser
    - Download the OpenAPI specification

    <Note>
    The interactive docs are powered by Swagger UI and are available at `/docs`. An alternative ReDoc interface is available at `/redoc`.
    </Note>
  </Step>
</Steps>

## Understanding the API

The `/analyze` endpoint is defined in `app/main.py:27` and accepts a simple request format:

### Request structure

```json
{
  "text": "string"
}
```

The `text` field is required and cannot be empty. If you send an empty string, you'll receive a 400 error.

### Response structure

```json
{
  "sentiment": "string",
  "score": 0.0
}
```

## How sentiment is determined

Behind the scenes, the API uses a machine learning pipeline:

1. **Text cleaning** (`app/preprocessing.py:4`) - Converts to lowercase, removes accents, strips special characters
2. **TF-IDF vectorization** (`app/model.py:10`) - Transforms text into numerical features
3. **Logistic Regression** (`app/model.py:11`) - Predicts sentiment with probability scores

The model returns probabilities for all classes and selects the one with the highest confidence.
